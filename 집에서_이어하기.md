# 집에서 이어서 작업하기

회사(세계일보)에서 하던 **세계일보 카드뉴스 생성 시스템** 작업을 집 PC에서 이어갈 수 있도록 정리한 문서입니다.

---

## 1. 폴더 위치

- **경로**: `c:\Users\KCHAN\OneDrive\바탕 화면\segye_cardnews_cursor`
- OneDrive 동기화로 회사·집 양쪽에서 같은 폴더를 사용할 수 있습니다.

---

## 2. 한 번에 실행 (권장)

1. **START.bat** 더블클릭  
   - 위치: `segye_cardnews_cursor` 폴더 루트
   - 백엔드(API) + 웹페이지가 **http://localhost:3000** 에서 함께 실행됩니다.

2. 브라우저에서 **http://localhost:3000** 접속

3. 기사 URL 입력 → **"AI로 생성하기"** 클릭 → 카드뉴스 생성

> **종료**: 실행한 명령 창에서 `Ctrl+C` 후 창 닫기

---

## 3. 환경 변수 (집 PC에서 처음 쓸 때)

API 키는 **회사 PC와 동일한 값**을 쓰면 됩니다. (키가 회사에만 있다면 회사에서 복사해 오거나, 동일 계정으로 재발급)

- **backend/.env** 파일이 있어야 합니다.
- 없으면 **backend/.env.example** 을 복사해 **backend/.env** 로 저장한 뒤, 아래 값만 채우면 됩니다.

| 변수 | 설명 | 발급처 |
|------|------|--------|
| **GEMINI_API_KEY** | Gemini API (우선 사용) | https://aistudio.google.com/apikey |
| **OPENAI_API_KEY** | OpenAI API (폴백/이미지 생성) | https://platform.openai.com/api-keys |

- **CORS_ORIGIN**: 기본값으로 `http://localhost:3000`, `http://127.0.0.1:3000` 등이 포함되어 있어, START.bat으로만 쓸 경우 수정하지 않아도 됩니다.

## 4. 프로젝트 상태 (참고)

- **통합 완료**: 실제 기사 크롤링(scraper), 카드뉴스 AI 생성(Gemini/OpenAI), 프론트 연동까지 구현됨.
- **문서**: `INTEGRATION_PLAN.md`(통합 계획), `QUICKSTART.md`(실행 요약), `ENV_AND_DEPS.md`(환경/의존성).
- **남은 작업**: 계획서 기준으로는 **전체 플로우 테스트**(URL 입력 → 생성 → 화면 표시)를 사용자 환경에서 확인하는 단계입니다.

---

## 5. 문제 해결

| 증상 | 확인할 것 |
|------|-----------|
| **'node' is not recognized** | 집 PC에 Node.js 미설치 또는 PATH 미등록. **Node설치_안내.md** 참고 후 설치하고, 명령 창을 새로 연 다음 START.bat 다시 실행 |
| "기사를 불러올 수 없습니다" | URL 확인, 해당 사이트 접속 가능 여부 |
| "AI 서비스 오류" | **backend/.env** 의 `GEMINI_API_KEY` 또는 `OPENAI_API_KEY` 값 |
| 포트 3000 이미 사용 중 | START.bat이 실행 시 기존 3000 포트 프로세스를 종료하도록 되어 있음. 그래도 문제면 작업 관리자에서 `node` 프로세스 확인 |
| CORS 오류 | 프론트를 다른 포트(예: Live Server 5500)로 열었다면 **backend/.env** 의 `CORS_ORIGIN`에 해당 주소 추가 (쉼표로 구분) |

## 6. 터미널로 실행하고 싶을 때

```bash

```
